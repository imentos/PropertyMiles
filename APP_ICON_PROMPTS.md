# Property Miles - App Icon Generation Prompts

## Requirements
- Size: 1024x1024 pixels
- Format: PNG with no transparency/alpha channel
- Style: Clean, professional, recognizable at small sizes (29x29 to 1024x1024)
- Must represent: Mileage tracking + Real estate
- Color scheme: Professional, stands out on iOS home screen
- No text in the icon

---

## Prompt 1: Car with Location Pin (Recommended)

```
Create a modern iOS app icon for a mileage tracking app. Design a minimalist car icon viewed from above in a clean line style, with a location pin marker overlaid or emerging from it. Use a gradient background transitioning from deep blue (#1E3A8A) at the bottom to teal (#0D9488) at the top. The car should be white or light colored for contrast. The location pin should be a bright accent color like orange (#F97316) or red (#EF4444). The design should be clean, professional, and easily recognizable at small sizes. No text. Centered composition. Flat design with subtle shadows for depth. iOS app icon style with rounded corners applied by the system.
```

**Design Elements:**
- Background: Gradient (blue to teal)
- Main icon: Simplified car (top view), white/light gray
- Accent: Location pin, orange/red
- Style: Flat design, minimal


## Post-Generation Checklist

After generating the icon:

- [ ] Verify dimensions: Exactly 1024x1024 pixels
- [ ] Check format: PNG, RGB color space
- [ ] Confirm no transparency: All pixels have 100% opacity
- [ ] Test at multiple sizes: 29x29, 60x60, 180x180, 1024x1024
- [ ] Verify visibility: Clear on both light and dark backgrounds
- [ ] Check contrast: Meets accessibility standards
- [ ] Ensure uniqueness: Not confusable with competitors (MileIQ, Everlance, TripLog)
- [ ] Review Apple guidelines: No screenshots, buttons, or rounded corners (system applies them)
- [ ] Get feedback: Show to 3-5 potential users
- [ ] Save original vector: Keep source file for future edits

---

## Apple Human Interface Guidelines

**Required:**
- Square format (system adds rounded corners)
- No transparency
- No alpha channel
- Solid backgrounds

**Recommended:**
- Centered composition
- Simple, recognizable imagery
- Avoid lots of detail (won't show at small sizes)
- Consistent with app's visual style
- Unique and memorable
- Appropriate for all audiences

**Avoid:**
- Photos or screenshots
- Text that's hard to read
- Multiple words
- Replicas of Apple hardware
- Apple interface elements
- Politically controversial imagery
